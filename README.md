# 会议开完了 · meeting-done

会议全生命周期管理自动化技能。一句话触发，从预约、转录、整理、归档到提醒的完整闭环。

## 功能概览

### 分支 A：会后整理
**触发语：** 「开完会了」「会议结束了」

```
自动拉取最近会议 → 获取智能纪要 + 转录原文
→ 展示纪要预览供你确认 → 创建文档 → 写入Excel索引表
→ 发送邮件给参会人 → 输出执行报告
→ 问你是否预定下次会议
```

### 分支 B：会前预约
**触发语：** 「帮我预定一个会议」

```
收集主题和时间 → 预约腾讯会议（自动开启转录）
→ 创建纪要文档占位 → 设置会前提醒
→ 提醒内容包含：本次会议 + 上次纪要 + 待对齐事项
```

## 前置依赖

| 依赖 | 说明 |
|------|------|
| 腾讯会议 MCP | 需已配置腾讯会议 API |
| 腾讯文档 | 用于会议纪要文档和索引表 |
| QQ邮箱 SMTP | 用于发送邮件 |
| 联系人库 | `memory-tdai/联系人.md`，姓名→邮箱映射 |

## 固定资源配置

在 `SKILL.md` 中修改以下配置：

```yaml
# === 腾讯文档 ===
EXCEL_FILE_ID: ""      # 腾讯文档 Excel 索引表的 file_id
EXCEL_SHEET_ID: ""      # 索引表的工作表 ID

# === 邮件 ===
SMTP_EMAIL: ""          # 你的发件邮箱地址
SMTP_SCRIPT: "~/Library/Application Support/QClaw/openclaw/config/skills/imap-smtp-email/scripts/smtp.js"

# === 联系人库 ===
CONTACTS_FILE: "memory-tdai/联系人.md"   # 参会人姓名→邮箱映射文件
```

## 文件结构

```
meeting-done/
├── SKILL.md      # 技能定义（AI 使用）
├── README.md     # 说明文档
└── scripts/      # 脚本目录（当前为空）
```

## 安装

将整个目录放入 OpenClaw skills 目录：

```bash
cp -r meeting-done ~/.qclaw/skills/meeting-done
```
